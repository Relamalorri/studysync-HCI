# Design Principles

## Feedback

### Task Completed Successfully
The "Add Task Complete" screen confirms to the user that their task was created successfully, providing clear system feedback after every action.

### Course Added Successfully
The "Add Course Complete" screen similarly confirms successful course creation, closing the feedback loop for the user.

---

## Visibility

### Highlighted Bubble for Current Page
In course pages, a dark purple highlighted bubble indicates which subpage (Material or Tasks) the user is currently viewing.

### Strikethrough on Completed Tasks
Completed tasks display a strikethrough on their title, giving users an immediate visual cue of their progress.

### Placeholder Text
Every input field contains placeholder text describing exactly what the user should enter — visible on Sign Up, Log In, Add Task, and Add Course screens.

---

## Constraints

### Character Limit on Notes
The Add Task Step 3 screen displays a "0/200 characters left" counter, informing users of input boundaries before they hit them.

---

## Consistency

### Unified Color Palette
Dark purple, light purple, and neutral greys are used consistently throughout all screens. Corner radius, font sizes, and button styles are uniform across the entire interface.

---

## Cognitive Processes

### Memory — Color Coded Courses
Each course is assigned a unique color that appears consistently across the dashboard, calendar, and progress screens, reducing the memory load of tracking multiple subjects.

### Attention — Red Log Out Button
The red color of the logout button in the Profile screen draws immediate attention and distinguishes it from other settings options.

### Perception — Content Grouping
The dashboard groups tasks under "Due Today" and "Upcoming" headers, allowing users to immediately perceive task urgency without reading every item.
